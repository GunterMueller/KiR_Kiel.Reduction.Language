Die Papersyntax von KiR

Der Editor wurde um eine Option und ein Kommando erweitert. Diese dienen dem Zweck, KiR-Programme in einer Form auszugeben, die näher an den allgemeinen Konventionen ist (sog. Papersyntax).

Das neue Kommando heißt 'paper' und gibt analog zu 'pp' den aktuellen Cursorausdruck in Papersyntax (siehe unten) aus.

Die neue Startoption heißt '-P', die analog zu '-p' dazu dient, die angegebenen Programme im Batchverfahren in Papersyntax auszugeben.

Das sonstige Verhalten des Editors ist unverändert.

Die Papersyntax

Die Papersyntax unterscheidet sich in folgenden Punkten von der normalen (Editor-)Syntax:
  • Schlüsselworte und primitive Funktionen werden groß geschrieben.
  • Variablen/Konstruktoren werden klein geschrieben.
  • Kommas zur Trennung von Listenelementen, Parametern etc. wurden entfernt.
  • Infix- wurde in Präfixnotation umgewandelt.
  • Es gibt nur noch eine Schreibweise (func arg1 .. argn) für Applikationen: 
    primf(arg1, .. , argn)          -> (primf arg1 .. argn)
func [ arg1, .. , argn ]        -> (func arg1 .. argn)
(arg1 func arg2)                -> (func arg2 arg1)
ap func to [ arg1, .. , argn ]  -> (func arg1 .. argn)
  • eckige Klammern um Parameterlisten von Funktionsdefinitionen entfallen.
  • Statt 'def ... in ...' wird 'LETREC ... IN ...' geschrieben.
  • Statt 'sub ... in ...' wird 'LAMBDA ... IN ...' geschrieben.

Beispiel:

Editorsyntax:

def 
   F [ X, Y ] = if ( 1 gt 0 )
                then < X, X, Y, Y >
                else sub [ X, Y, Z ]
                     in Y,
   Apply [ FL, X, Y ] = ap ( 1 | FL )
                        to [ X, Y ]
in lselect( 2 , Apply [ < F, G >, Y, X ] )

Paper-Syntax:

LETREC 
      f x y = IF ( GT 0 1 )
              THEN < x x y y >
              ELSE LAMBDA x y z 
                   IN y
      apply fl x y = ( ( | fl 1 ) x y )
IN ( LSELECT 2 ( apply < f g > y x ) )
